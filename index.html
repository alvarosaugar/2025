Mes de QA

Es una iniciativa donde se anima a que, durante un mismo mes, distintas comunidades puedan organizar y promocionar charlas y encuentros sobre esa temática, y dotar de un punto para la promoción cruzada de esas charlas y comunidades.

¿Más información? ¿Ediciones anteriores y más? Accede a la web del Mes de QA.

10/06/2025

19:00 (UTC +2) presencial Top Security Flaws Hiding in Your Code Right Now – and How to Fix Them

Esta sesión será en español

Security vulnerabilities aren’t just hypothetical—they’re real, from the MongoDB attack to the Log4Shell disaster, and they’re hiding in your code right now. In this session, we’ll explore the most common and dangerous flaws: SQL Injection, Deserialization Injection, and Logging Injection.

We’ll explain how these vulnerabilities work, why they’re so risky, and how to fix them through examples.

You'll leave with practical tips for safeguarding your applications, including best practices for SQL query parameterization, secure deserialization techniques, the importance of sanitizing logs, and how to use tooling to prevent these flaws more easily.

Audience Takeaways:

Deep understanding of SQL Injection, Deserialization Injection, and Logging Injection. Practical code examples and fixes to secure your applications. Insights into real-world case studies of major security breaches. Tooling that will help us to commit safe code.

FAQ

Objetivo que tiene Mes de QA

Dar visibilidad a QA y establecer relaciones entre ponentes y comunidades que tratan el tema.

De quién son las sesiones

Las sesiones son de las comunidades o la persona que la dé, por lo que son ellas las que deciden y gestionan (podemos ayudar si se necesita).

Requisitos para participar

Sólo se piden dos requitos:

  1. La temática de la sesión debe ser QA;
  2. Se anuncia la iniciativa y comparte la web al comienzo de la sesión para dar visiblidad al resto de participantes
